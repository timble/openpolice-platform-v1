Streaming Remote Content

AllVideos Reloaded supports easy shotcuts for embedding media from popular video sites. Please note: All those sites provide their own player which is just embedded in your web site. Those "foreign" players normally only support setting the witdh and height of the video. Some players also support control over autostart. All other parameters are are effective only, if the builtin player for FLV, SWF or MP3 media is used (i.e: for streaming of local media). The following Table shows all supported presets (in alphabetical order) and instructions on how to use them:

Presets, marked with a red star (*), use a special ripper technique in order to circumvent URL obfuscation, used by the provider: They first request a page from the provider, parse the response in order to get the real URL of the video file and then create the player markup according to the data retrieved from that response. Therefore, embedding videos from those providers might slow down the response of your site on the embedding page. Since version 1.0.beta3, a cache is used for storing the retrieved data, so after invoking the corresponding page at least once should give a significant performance boost. Also, for embedding those providers, your server needs permission to establish outgoing connections. Most hosters allow that, however if you get a message like "Unable to fetch URL: ..." instead of the video, then such a restriction might be in place and you should contact your hoster.
