Remote Inhalte

AllVideos Reloaded unterstützt einfache Kürzel zum Einbetten von Medien auf populären Video-Sites. Bitte beachten Sie, dass alle diese Sites ihren eigenen Player bereitstellen. Dieser wird dann jeweils in Ihrer Seite eingebettet. Diese "fremden" Player unterstützen meist nur wenige Parameter wie z.B. das Setzen der Breite und der Höhe des Videos. Einige unterstützen zusätzlich auch autostart. Alle darüber hinaus gehenden Parameter haben nur Wirkung, wenn der eingebaute Player für FLV, SWF oder MP3 Median benutzt wird (zum Streamen von lokalen Inhalten). Die folgende Tabelle zeigt die unterstützten Tags resp. Presets (alphabetisch sortiert) sowie Instruktionen zur Anwendung spezifischer Tags:

Presets, welche mit einem roten Stern (*) markiert sind, verwenden eine spezielle ripper Technik um die URL-Verschleierung des jeweiligen Providers zu umgehen: Hierbei wird zuerst die relevante Seite des Providers abgerufen und deren Inhalt dann im Plugin geparst um an die tatsächliche URL des Videos zu gelangen. Danach wird dann der für das Einbetten nötige Markup-Code generiert. Aus diesem Grund, können von diesen Providern abgerufene Videos den Seitenaufbau Ihrer Site verzögern. Seit Version 1.0beta3 des Plugins sorgt ein eingebauter Cache dafür, dass diese Verzögerungen auf ein absolutes Minimum reduziert werden. Nach dem erstmaligem Abrufen einer solchen Seite sollte sich also ein signifikanter Geschwindigkeitsvorteil einstellen. Darüberhinaus, erfordert das Einbetten von Medien solcher Provider natürlich den Aufbau einer ausgehenden Verbindung von Ihrem Server zum jeweiligen Video-Provider. Die meisten Hoster haben die Server so eingestellt das dies erlaubt ist. Falls jedoch anstelle des Videos eine Meldung "Unable to fetch URL: ..." erscheint, dann könnte es sein dass Ihr Hoster dies nicht zulässt. In diesem Fall kontaktieren Sie bitte Ihren Hoster.
