Afspeellijsten gebruiken

Wanneer je lokale media streamt met de JW Media speler, kun je in plaats van een enkel media bestand, gebruik maken van een XML afspeellijst bestand. Natuurlijk, als je wilt voorzien in een afspeellijst bestand, dien je gebruik te maken van de auto tag of één van de remote varianten, omdat deze als enigen geen gebruik maken van enige automatisch voorgedefinieerde extensies. In plaats van gebruik te maken van een directe media url, kun je nu de afspeellijst URL gebruiken. De speler ondersteund drie algemeen gebruikte afspeellijst formaten om zeker te kunnen zijn van maximale verenigbaarheid: XSPF (veel gebruikt voor CC muziek), RSS (veel gebruikt voor Podcasts) en ATOM (veel gebruikt door Blogs). Op Jeroen's website, heeft hij een aantal extra voorbeelden geplaatst van een xspf afspeellijst en een rss afspeellijst, met bijna alle ondersteunde tags inbegrepen.

Met afspeelijsten, kun je enkele effecten bereiken die niet beschikbaar zijn met het gebruik van de standaard media URLs. Bijvoorbeeld, je kunt een vooruitblik afbeelding toevoegen aan een video. Deze afbeelding is zichtbaar, ook wanneer de speler is gestopt. Hier is een simpel voorbeeld. Dit voorbeeld word gebruikt op de website om een vooruitblik afbeelding te tonen in de kleine video op de voorpagina:

 
  <playlist version="1" xmlns="http://xspf.orig/ns/0/">
    <title>CG Playlist</title>
    <trackList>
      <track>
        <title>Reloaded</title>
        <location>/images/stories/videos/matrix.flv</location>
        <image>/images/stories/videos/matrix_thumb_small.jpg</image>
      </track>
    </trackList>
  </playlist>
 

Please note also, that if you use the auto tag (without an absolute URL), then the playlist file itself must be located in the images/stories/videos subdir, even if the list references only audio files. Furthermore, if you use a .m4v file extension in the location tag, then you must specify the media type manually for the relevant track by adding the following element at the end of the track definition:<meta rel="type">flv</meta>

Afspeellijsten Tonen

Een behoorlijke afspeellijst tonen kan soms een gevecht worden als je niet weet hoe de speler omgaat met de verschillende parameters binnen je inhoud. Hier zal ik proberen de logica uit te leggen: Voor beiden, de speler zijn breedte en hoogte, bestaat een tweede parameter genaamd vertoonbreedte en vertoonhoogte. Normaliter worden deze niet gedefinieerd met als resultaat dat de speler de standaard afbeelding laat zien. Indien één van beiden wel is ingesteld als kleinere waarde, zal de afspeellijst getoond worden. Als je de vertoonbreedte instelt, zal de afspeellijst vertoont worden aan de rechterzijde van het videofame boven het controle paneel. De breedte van de afspeellijst is simpel breedte - vertoonbreedte in dat geval. Het gebruik van vertoonhoogte is een beetje meer gecompliceerd, omdat voor het controle paneel gerekend moet worden. De hoogte van het controle paneel is 20px, dus als je de vertoonhoogte instelt op een kleinere waarde dan hoogte - 20, zal de afspeellijst dat verschil opvullen onder het controle paneel. In dit geval is de hoogte van de afspeellijst hoogte - vertoonhoogte - 20. Er is ook een speciale kwestie waar geen afspeellijst word getoond maar een ander effect word bereikt: Het instellen van de vertoonhoogte gelijkwaardig aan de hoogte zorgt ervoor dat het controle paneel een beetje kleiner word binnenin de video en komt te voorschijn/verdwijnt wanneer de muis over de video heengaat. Verder, indien je een .m4v bestandsextensie gebruikt in de lokatie tag, dan moet je het media type handmatig specificeren voor de relevante track door het volgende element op het einde van de track definitie: <meta rel="type">flv</meta>

Parameters

Sommige parameters van de speler zijn gerelateerd aan afspeellijsten: Als je de afbeeldingen wilt tonen die toegevoegd zijn aan de afspeellijst ingangen, in de lijst als thumbnails getoond, kun je de parameter plthumbs op waar instellen.
